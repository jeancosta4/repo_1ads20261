# 📘 Guia Completo de Markdown (GitHub)

Este documento apresenta os principais comandos e recursos da linguagem **Markdown**, amplamente utilizada em documentação, README de projetos e anotações técnicas.

---

# 🧱 1. Títulos (Headings)

```md
# Título H1
## Título H2
### Título H3
#### Título H4
##### Título H5
###### Título H6
```

---

# ✍️ 2. Formatação de Texto

```md
**Negrito**
*Itálico*
***Negrito e Itálico***
~~Texto riscado~~
```

**Resultado:**

* **Negrito**
* *Itálico*
* ***Negrito e Itálico***
* ~~Texto riscado~~

---

# 📌 3. Listas

## 🔹 Lista não ordenada

```md
- Item 1
- Item 2
  - Subitem
```

## 🔸 Lista ordenada

```md
1. Primeiro
2. Segundo
3. Terceiro
```

---

# ✅ 4. Lista de Tarefas (Checklist)

```md
- [x] Tarefa concluída
- [ ] Tarefa pendente
```

---

# 🔗 5. Links

```md
[Texto do link](https://github.com)
```

---

# 🖼️ 6. Imagens

```md
![Texto alternativo](https://via.placeholder.com/150)
```

# 💻 7. Código

## 🔹 Código inline

```md
`console.log("Olá mundo")`
```

## 🔸 Bloco de código

<pre>
```javascript
function hello() {
  console.log("Olá mundo");
}
```
</pre>

```python
print('Olá, mundo!')
```

# 🧠 14. Emojis (GitHub)

Você pode usar emojis diretamente:

```md
🚀 🔥 📘 ✅
```

Ou usando códigos:

```md
:rocket: :fire: :book:
```

# 📍 15. HTML dentro do Markdown

O Markdown aceita HTML:

```md
<b>Negrito com HTML</b>
<br>
<i>Itálico com HTML</i>
```

# 📦 17. Estrutura de README Profissional

Exemplo:

```md
# Nome do Projeto

## 📌 Descrição
Descrição do projeto

## 🚀 Tecnologias
- HTML
- CSS
- JavaScript

## 📦 Instalação
Passos para rodar o projeto

## 👨‍💻 Autor
Seu nome
```

# ⚠️ 18. Dicas Importantes

* Markdown é sensível a espaços
* Use linhas em branco para melhor organização
* GitHub possui variações próprias (GFM - GitHub Flavored Markdown)
